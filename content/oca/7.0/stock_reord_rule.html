Improved reordering rules

Name Improved reordering rules stock_reord_rule
Author Sergio Corato
Website http://www.icstools.it
Version 0.2
Dependencies procurement, sale
Repository stock-logistic-warehouse
  • Description

This module allows to improve reordering rules of stock module.

It works forecasting the stock needed per product for n days of sales, with the next formula: (( Qty sold in days_stats * (1+forecast_gap)) / days_stats * days_warehouse) where: - days_stats = days on wich calculate sales stats; - forecast_gap = forecast of increase/decrease on sales (%); - days_warehouse = days of stock to keep in the warehouse.

Usage: insert days_stats, forecast_gap and days_warehouse vars in product form and create a reordering rule for the same product, without inserting nothing (neither maximum or minimum quantity are required). The cron job will be executed daily and will update the maximum quantity in the reordering rule (you can force it to start changing the date and hour of execution). This module doesn't need purchase module to work, but it's useful with that module.'
